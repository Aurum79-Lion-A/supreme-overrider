## Reporting a Vulnerability

Security is a top priority for this project. If you have discovered a security vulnerability, please do not report it in a public issue. Instead, please report it to us privately via e-mail so that we can address it before it is exploited.

How to Report
To report a security vulnerability, please use the following method:

Email: [mrcobalt.info@gmail.com]

Alternative: [If applicable, mention using GitHub's "Private vulnerability reporting" feature]

Please include the following in your report:

A brief summary of the vulnerability.

Steps to reproduce the issue (Proof of Concept - PoC).

The specific version(s) of the project affected.

What to Expect
After you submit your report, here is the process you can expect:

Initial Response: We aim to provide an initial response within [e.g., 48-120] hours.

Updates: We will provide you with updates on the status of the investigation at regular intervals (e.g., weekly).

Outcome:

If accepted: Once the vulnerability is verified, we will begin working on a fix and will notify you as soon as it is ready for deployment.

If declined: If the reported issue is not considered a security vulnerability, we will explain our reasoning and provide additional details regarding the project's security posture.

We appreciate your responsible disclosure and thank you for helping us keep this project secure.
